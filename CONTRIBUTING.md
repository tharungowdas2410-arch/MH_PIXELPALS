# Contributing to VeriStream

This document provides minimal guidance for making internal changes.

## Development

1. **Set up environment**:
    Copy the example environment file:
    ```bash
    cp .env.example .env
    ```

2. **Run with Docker**:
    ```bash
    make start
    ```

3. **Run Tests**:
    ```bash
    make test
    ```

## Style

- Follow existing code patterns and naming conventions.
- Keep changes small and focused.
